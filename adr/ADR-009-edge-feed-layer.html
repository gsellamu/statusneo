Architecture Decision Record · ADR-009

Renderer-agnostic twin, rendering-optimized edge layer

The twin stays the single source of truth and rendering-agnostic. A thin edge/feed layer shapes that same event-sourced truth for each renderer's latency and bandwidth profile — never forking truth, never embedding rendering logic in the twin.

Status · Proposed Date · 2026-06-14 Deciders · CTO · Lead Architect · Platform pod Anchor · four-tier latency model
Decision. Keep the twin rendering-agnostic. Insert rendering-optimized edge/feed adapters between the twin and each renderer that shape the same event-sourced truth per consumer — not separate twins, just differently-shaped reads of one event log.
One twin, many edges, many lenses. Truth is shaped at the edge, never forked.
1Context — what exists, why it doesn't scale

Today every lens talks directly to one GraphQL endpoint and pulls the same full payload twin(room){version instances bindings}. Five renderers (2D canvas, Schematic SVG, 3D Three.js, Twin-Data table, Unity) reconstruct the room from instances[] + bindings[] with zero server-side rendering code. That proves the thesis — the renderer owns nothing, the twin owns the truth — and is the correct demo simplification. It does not scale: one payload shape is forced on every consumer (bytes for a table vs. megabytes of mesh for Unity), and a single central endpoint cannot do the edge prediction that actually kills felt latency.

2The four-tier latency model (the anchor)

The edge/feed layer is precisely the apparatus that delivers T1 (optimistic + edge confirm) and T0-friendly content delivery. T2/T3 stay twin-side.

T0< 16 msNever networked — pure UnityCamera, hover, selection, drag preview

3Three movement mechanisms, deliberately separated

Content is cached · state is synced · the brain is relocated. Three different problems, three different mechanisms — never one pipe.

Geometry / mesh

Edge CDN · immutable

Render meshes (glTF / USD) are immutable and content-addressed — hash is identity. Served cache-forever from a CDN. Unity / WebGL pull a bundle once; reuse costs zero bytes.

The twin never streams triangles. It emits a reference (typeId + version → hash); the renderer resolves it. Light lenses ignore it.

Mutable twin state

CQRS snapshot + delta

The event-sourced twin is read as a snapshot plus a delta stream, not repeated full polls. Same truth, different envelope per edge — a web lens takes a small JSON diff; Unity may take the same delta over a binary channel.

A read-model projection, not a second truth. Exactly one event log. Regional read replicas serve snapshots locally; the write path stays central.

The brain (LLM + GraphRAG)

Co-located regional

Keep retrieval and inference together and near the data — a cross-region GraphRAG round-trip costs more than the inference. The local-first posture already is edge inference.

Never on the interaction path. Proposals stream as T2 overlays; they never block T0/T1. It snaps instantly; it certifies asynchronously.
4Optional AEC differentiator — the jobsite edge node

A small box in the construction trailer runs the viewer payload, a read snapshot, and a local validation worker. It serves an offline-first viewer and confirms optimistic placements locally, syncing to the regional write path when the link returns. Offline-first becomes a product differentiator, not a slide.

5Invariants — the guardrails truth never crosses
  1. One event log. One source of truth. Edges are projections/caches, never authorities. No edge accepts a write it has not forwarded to the twin.
  2. No rendering logic in the twin. It emits typed truth (instances, bindings, geometry references). It does not know what a triangle is.
  3. No truth in the renderer. Unity / WebGL hold ephemeral view state only. The edge does not change that.
  4. Optimistic ≠ authoritative. An edge-confirmed placement is provisional; the twin's committed verdict replaces it. The deterministic gate still disposes.
  5. Content-addressed immutability. A mesh bundle is identified by hash, never mutated in place. Cache invalidation is therefore a non-problem.
6Consequences

Positive

  • Each consumer gets the lightest correct shape: bytes for a table, mesh-once for Unity.
  • Geometry caching + optimistic confirm kill felt latency at T0/T1 without moving the LLM.
  • Offline-first becomes real (jobsite node).
  • The renderer-independence thesis is preserved and strengthened.

Cost

  • Operational surface grows: CDN, read replicas, delta projection, edge workers.
  • A read-model projection must track the event log (a bounded eventual-consistency window on reads).
  • More to monitor — each edge needs health + lag metrics.

Neutral

  • The current direct-to-GraphQL path is not discarded — it is the degenerate case (one edge = the origin).
  • The demo keeps working; the edge layer is additive and reversible.
7Rollout — incremental, gated, no big bang
Phase 0 · today

Direct-to-GraphQL. Proven. Ship the demo on this.

Phase 1

CQRS snapshot + delta read model behind the existing subscription. No client change.

Phase 2

Geometry → content-addressed CDN bundles; twin emits references. Light lenses unaffected.

Phase 3

Regional read replicas + co-located brain.

Phase 4

Jobsite edge node — offline-first viewer + local validation worker.

Each phase is independently shippable and reversible. Truth never forks at any phase.

Unity becomes the GPU. The twin holds the truth. The edge carries the bytes. The brain stays where the data is.
