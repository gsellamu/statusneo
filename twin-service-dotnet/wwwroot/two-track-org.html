1+2The two tracks

Track 1 designs the technical organization (who owns which part of the platform). Track 2 designs the execution organization (how work flows, with AI in the loop). Both report through one FDE.

Track 1 · Technical Org

Platform ownership by value stream

The shape

  • FDE / Platform Architect (onsite) — single point of technical ownership; owns the architecture of record, the ADRs, and the twin/agent/gate spine.
  • Offshore execution pods — Platform, App, UX, Unity (3D), Data & AI, Cloud — each a bounded value stream with clear authoritative-for boundaries.
  • Technical Engineering Anchor (offshore) — holds SDLC discipline and quality bars across pods.

What it fixes

  • Distributed ownership → one named owner per value stream.
  • Truth split across Unity + batch pipeline → one twin owns truth; Unity renders.
  • No AI/agentic layer → governed agentic AI + inline gate, owned by the FDE.
Track 2 · Execution Org

AI-augmented delivery, outcome-led

The shape

  • Engineering Lead over Scrum Master + Product Owner, driving roadmap-based iterative delivery.
  • AI-DLC loops, not projects: Define → Design → Engineer → Automate → Govern, continuously running.
  • Super-agents in the SDLC — planning, PR review, test generation, QA — augmenting humans, never replacing the gate.

What it fixes

  • Task-driven → outcome-led engineering.
  • Manual SDLC → AI-augmented SDLC (Spec → BDD → ATDD).
  • Bottom-up planning → roadmap-based, predictable 2-week cycles.
