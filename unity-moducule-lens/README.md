# Unity Moducule Lens — Modutecture's own engine, as a 4th lens over the live twin

A complete, openable **Unity 6.1 (6000.1.14f1)** project that renders the Modutecture
twin and emits PLACE intents — the same contract as the web lenses (2D / Schematic /
3D / Twin Data). It proves the thin-Unity thesis with Modutecture's *own* engine:
**the engine renders; the twin owns truth.** "Unity becomes the GPU; the twin holds the truth."

Two run shapes from ONE shared C# core:
- **Native player** — runs beside the browser (your ResilientMind posture). Live WebSocket push.
- **WebGL build** — embedded as the studio's Unity lens (iframe). Polling fallback.

---

## Architecture (why this is defensible to the Chief Architect)

The client holds **no domain state**. Every twin payload is a fresh projection; placement
goes to the spine's deterministic gate, which owns truth. Swap this engine for the web
lenses and nothing changes server-side — identical GraphQL contract. The engine is *a
lens*, not the source of truth. This is the same shape as the ResilientMind UE5 client:
**versioned payloads down, intents up, telemetry in.**

```
Spine (.NET, :5005)  ──GraphQL twin──▶  TwinClient ──▶ RoomRenderer (prefabs)
        ▲                                   │
        └────────── PLACE intent ───────────┘   (gate validates; next payload renders)
```

Transport: **WebSocket-primary** (`graphql-transport-ws` subscription) with automatic
**polling fallback**. Native gets live push; WebGL (no `System.Net.WebSockets`) and any
socket failure degrade to 1s polling. Either way the twin renders.

## Run it — native (the hero demo)

1. **Start the spine** (from the repo root, top-level — NOT the nested duplicate):
   ```powershell
   cd twin-service-dotnet
   dotnet run
   ```
   Confirm the URL it prints (see "Port note" below). Open it; the web lenses should work.

2. **Open the Unity project**: Unity Hub ▸ Add ▸ select `unity-moducule-lens/`.
   Open with **6000.1.14f1**. First open resolves packages + imports (~1–2 min).

3. **Build the scene**: menu **Modutecture ▸ Build Moducule Lens Scene**.
   This generates the floor, camera, lighting, 3 color-coded Moducule prefabs, and wires
   `TwinClient` + `RoomRenderer` + `IntentEmitter`. Re-runnable.

4. **Press Play.** The lens polls/streams the twin and renders. **Right-drag** to look,
   **scroll** to zoom, **left-click the floor** to place the selected Moducule (default
   `bed-icu`) — it routes through the gate; the next payload renders the result, and every
   web lens updates too.

> If your spine runs on a port other than 5005, set it on the `TwinClient` component in
> the Inspector (`httpUrl` / `wsUrl`) before pressing Play, or call Configure (below).

## Run it — WebGL lens (portable, embedded in studio.html)

1. **Build**: menu **Modutecture ▸ Build WebGL into Spine wwwroot**. It switches the
   target to WebGL (one-time, slow), selects the `ModuLens` template, and outputs to
   `twin-service-dotnet/wwwroot/unity/`.
2. Open `http://localhost:5005/studio.html`, tick the **Unity** lens. studio.js probes
   `/unity/index.html`; finds the build; embeds it; and pushes config
   (`httpUrl|wsUrl|room`) to it on ready.
3. Placing in the Unity lens posts an intent message up; studio.js refreshes all lenses.

## The host ↔ Unity bridge (already wired)

- **Unity → host**: `ModuBridge.jslib` posts `{source:"modutecture-unity",type:"intent"}`
  on every commit; studio.js refreshes all lenses in lock-step. On boot it posts `…type:"ready"`.
- **host → Unity**: studio.js posts `{source:"modutecture-host",type:"configure",
  spec:"httpUrl|wsUrl|room"}`; the template relays it via
  `unityInstance.SendMessage("TwinClient","Configure",spec)`.

CORS is open on the spine (`AllowAnyOrigin`), so the WebGL build can call `/graphql`.

## Port note (resolve before the demo)

The web lenses are served by the spine. Earlier work used **:5005**; the repo's
`QUICKSTART-WINDOWS.md` says **:5000**. Check what `dotnet run` actually prints and:
- The Unity **WebGL lens** auto-adapts (it derives URLs from `location.origin`).
- The Unity **native** client defaults to `:5005` — change `httpUrl`/`wsUrl` on the
  `TwinClient` component if your spine differs.
